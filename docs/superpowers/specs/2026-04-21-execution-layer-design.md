# DeFi Agent Execution & Adapter Layer - Design Specification

## Overview
This specification outlines the architecture for the **Execution Layer**, providing the DeFi AI Agent with the ability to interact with real blockchain smart contracts. It moves beyond simple "wrappers" into a robust, production-grade **Transaction Management System** and a standardized **Protocol Adapter Engine**.

## 1. The Transaction Manager (`manager.py`)
Responsible for the safe and reliable delivery of transactions to the blockchain.
*   **Nonce Synchronization:** Maintains an internal counter and cross-references with the node to prevent "nonce-too-low" errors or gaps.
*   **Dynamic Gas (EIP-1559):** Implements a gas strategy that polls for current `baseFee` and `priorityFee`, with a "P90 aggression" setting for critical exits.
*   **Retry with Exponential Backoff:** Automatically re-signs and resubmits transactions if they are stuck in the mempool (price bumping) or if the node fails.
*   **Transaction Lifecycle Tracking:**
    *   `PENDING`: Broadcasted, waiting for inclusion.
    *   `CONFIRMED`: Included in a block (checks for `status: 1`).
    *   `REVERTED`: Failed execution (extracts error reason for AI Brain analysis).

## 2. The Adapter Engine (`engine.py`)
A unified interface that decouples the AI Brain from protocol-specific complexities.
*   **Generic Action Interface:**
    ```python
    execute(action: ActionType, params: dict) -> TxData
    ```
*   **Action Types:** `DEPOSIT`, `WITHDRAW`, `SWAP`, `APPROVE`.
*   **Standardization:** Ensures that inputs (amounts, decimals) and outputs (transaction payloads) are consistent regardless of the target protocol.

## 3. Protocol Adapters (`adapters/`)
Specific implementations for each DeFi protocol.
*   **BaseAdapter:** Abstract class defining required methods and common utilities (e.g., ERC20 `approve` logic).
*   **AaveV3Adapter:** Handles interactions with Aave's `Pool.sol` (Supply, Withdraw).
*   **UniswapV3Adapter:** Handles interactions with `SwapRouter.sol` (ExactInputSingle, path routing).

## 4. Environment & Safety
*   **Connectivity:** Uses `Web3.py` for JSON-RPC communication.
*   **Secrets Management:** 
    *   `WALLET_ADDRESS` and `PRIVATE_KEY` loaded from `.env`.
    *   `.gitignore` updated to strictly exclude `.env`.
*   **Network Target:** Ethereum Mainnet (via Anvil Fork or Alchemy/Infura).

## 5. Implementation Units
*   `src/defi_agent/execution/manager.py`: The Shield (Tx handling).
*   `src/defi_agent/execution/adapters/base.py`: The Interface.
*   `src/defi_agent/execution/adapters/aave_v3.py`: Aave Implementation.
*   `src/defi_agent/execution/engine.py`: The Dispatcher.

## 6. Validation Policy (The Anvil Rule)
All execution code must be validated against a **Mainnet Fork (Anvil)**. 
*   **Test Criteria:** A transaction must go from "Generated by Brain" -> "Processed by Adapter" -> "Signed by Manager" -> "Confirmed by Anvil" without manual intervention.
*   No strategy profit assumptions are made during this stage; it is purely for **Execution Fidelity**.
